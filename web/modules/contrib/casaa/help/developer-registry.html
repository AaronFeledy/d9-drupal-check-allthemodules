The plug-in registry is where a plug-in module is defined to CASAA. This registry let's the framework know what plug-in modules it has available to include in its processes. The two functions needed to register the plug-in in the framework are:
  • casaa_register
  • casaa_unregister
These functions should be placed within the *.install file for your plug-in module.

Breakdown

casaa_register - (required) register the plug-in with the framework.
Parameters:
  • $plugin - an array keyed with the module name and the title of the plugin.
    • module - the module file's name
    • name - the human readable name for the module.
casaa_unregister - (required) remove the plug-in module's record from the registry.
Parameter:
  • The module file's name.
Example implementation: 
/**
 * Implementation of hook_install()
 * 
 * Send basic info about the plug-in to CASAA to
 * register the plug-in with the framework.
 */
function example_plugin_install() {
	$plugin = array();
	$plugin = array(
		'module' => 'example_plugin',
		'name' => 'Example',
	);
	
	casaa_register($plugin);
}

/**
 * Implementation of hook_uninstall()
 * 
 * Remove the plug-in from the registration from
 * the framework registry.
 */
function example_plugin_uninstall() {
	casaa_unregister('example_plugin');
}
